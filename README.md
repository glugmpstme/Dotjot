## Overall description
### Product perspective

The service, once deployed, will only have a web interface. This website will be utilized for execution of the provided functionalities.
From the member point of view, he/she will be able to register for the service through the website and also view the curriculum for the registered services.

### User characteristics

| User | Characteristics |
| --- | --- |
| Member | <ul><li>Ability to read and comprehend English</li><li>Possess a device to access the service</li><li>Predominantly interact with the service through Mobile device</li><li>Usage frequency is medium</li></ul> |
| Partner | <ul><li>Ability to read, understand and write English.</li><li>Familiarity with the operation of the basic Graphical User Interface (GUI).</li><li>Do not have a high technical enterprise</li><li>Possess a device to access the service</li><li>Usage frequency is low</li></ul> |
| Coordinator | <ul><li>Rudimentary understanding of modelling techniques.</li><li>Possess a device to access the service</li><li>Access to Database</li><li>Usage frequency is high</li><li>Predominantly interact with the service through a desktop</li></ul> |

### Constraints, assumptions and dependencies

1. All the users have access to a desktop/laptop/mobile device to access the service.
2. All users have an internet connection.
3. GUI is only in English.
4. Only registered coordinators have access to the service.

### Logical database requirement

Member Data entity:

Data Item | Type | Description
--- | --- | ---
**UUID** | Integer | Unique identifier generated by the database
First name | String | First name of member
Last name | String | Last name of member
Date of birth | Date | Date of birth of member
Sex | Character | Male or Female
Batch | Integer | Batch
SAP ID | Integer | SAP id provided y the college
Division | Character | Division member belongs to
Year | Integer | Course year the member belongs to
Course | String | Course the member is a part of
Stream | String | Stream
Email | Alpha-numeric | Email id of the member
Phone | Integer | Phone number of the member
Address | Alpha-numeric | Home address of the member


Partner Data entity:

Data Item | Type | Description
--- | --- | ---
**Partner_id** | Integer | Unique id of the partner
Name | Character | Name of the partner
Contact | Integer | Contact number of the partner organization
Email | Alpha-numeric | Email id of the partner organization
Coordinator name | String | Name of the coordinator from the organization
Location | String | Location(s) of the partner organization
Address | Alpha-numeric | Address of the location(s)
Specialization | String | Specialities the partner organization expects
Number of seats | Integer | Number of volunteers requested
Working days | String | The days on which Partner organization requires volunteers
Time slots | Time | Time slots along the duration of the working day
Seats per slot | Integer | Number of volunteers per slot
Remark | String | Any special requirements by the partner organization

NGO-selection Data entity:

Data Item | Type | Description
--- | --- | ---
**UUID** | Integer | Member's Unique identifier
Partner name | String | Name of the Partner
Time slots | Time | Service time slots
Days of visit | String | Service Days

Payment-confirmation Data entity:

Data Item | Type | Description
--- | --- | ---
**UUID**  | Integer | Member's Unique identifier
Payment-confirmation | Binary | To confirm payment of registration fees

Curriculum data entity:

Data Item | Type | Description
--- | --- | ---
**Partner_id** | Integer | Partner's Unique identifier
Curriculum | nVarchar | Stores Curriculum
